# London Underground Knowledge Graph (5CCSAKNE CW2)

a.k.a. *public-transport-knowledge-graph*

A London Underground knowledge graph project built with Python and RDF. The repository includes:

- A TBox ontology generator using `rdflib`.
- A structured data pipeline that consumes the TfL API.
- Documentation and prompt orchestration for ontology design.

## Key files

### Modelling

- `ontology_builder.py`: builds the TBox ontology and serialises it to Turtle
- `CQ.txt`: original competency questions for the domain

### Data Pipeline

- `pipeline_structured.py`: fetches TfL data, maps it to the ontology, and writes RDF instances
- `pipeline_text.py`: enriches the RDF graph with Wikipedia-derived transport metadata and text-based disruption analysis
- `pipeline_llm.py`: enriches disruption events using OpenAI extraction outputs
- `pipeline.py`: main entry point that combines structured and textual pipelines

### RAG pipeline

- `pipeline_rag.py`: identies and fills gaps in the knowledge graph
- `count_rag_triples.py`: counts the number of additional triples added by RAG

### SPARQL

- `sparqls.txt`: sparql queries
- `sparql_results.txt`: sparql queries output log
- `run_sparqls.py`: runs SPARQL queries on complete Turtle file
- `queries/`: contains sparql queries and results

### Misc

- `kXXXXXXXX_prompts.txt`: prompt engineering and modelling workflow used for ontology design. **NOTE: This is the final submission-led prompts file, renamed for each member in the group.**
- `merge_ontologies.py`: merges ABox and TBox files into a single Turtle file
- `ontologies/`: output Turtle files and ontology artifacts. (see [Output](#output))

## Dependencies

- Python 3.10+ (recommended)
- `rdflib`
- `requests`
- `openai`
- `python-dotenv`

## Usage

1. Install dependencies:

```bash
python -m pip install -r requirements.txt
```

2. Generate the ontology:

```bash
python ontology_builder.py
```

3. Set the TfL API key:

**Mac/Linux:**

```bash
export TFL_API_KEY=your_key_here 
```

**Windows:**

```bash
set TFL_API_KEY=your_key_here 
```

4. Run the structured data pipeline:

```bash
python pipeline.py
```

5. Run RAG completion

```bash
python pipeline_rag.py
```

6. Merge TBox and ABox Turtle files

```bash
python merge_ontologies.py
```

7. Validate with SPARQLs

```bash
python run_sparqls.py
```

## Output

- `ontologies/tbox_ontology.ttl`: Final modelled ontology TBox
- `ontologies/instances.ttl`: Merged knowledge graph instances (ABox) produced by the pipeline
- `ontologies/rag_completions.ttl`: Missing triples identified and generated by RAG completion
- `ontologies/instances_completed.ttl`: Final ABox Turtle file with RAG completion

- `ontologies/full_knowledge_graph.ttl`: The final Turtle file with TBox and ABox. **NOTE: This is the final submission-led TTL file. i.e. Equivalent to kXXXXXXXX_kg.ttl.**

## Notes

- `pipeline_structured.py` expects `TFL_API_KEY` in the environment.
- `pipeline_llm.py` expects `OPENAI_API_KEY` in the environment when using the LLM enrichment pipeline.
- The project uses the namespaces `http://example.org/ontology-express#` (EX) and `http://example.org/instances#` (INST).
- These are defined once in `pipeline_common.py` and imported wherever needed, ensuring consistency across all scripts.
- There are no local EX or INST definitions in individual scripts; all scripts use the shared constants from `pipeline_common.py`.
- The design is intended to automate knowledge graph construction rather than rely on manual graph editing.
